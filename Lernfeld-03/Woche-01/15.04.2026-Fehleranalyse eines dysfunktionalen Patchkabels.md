# 1. Warum ist die Verdrillung der Aderpaare so entscheidend?
Problem ohne Verdrillung (parallel geführte Einzeladern)
Bei parallel geführten Leitungen wirken die Adern wie Antennen:

Sie fangen elektromagnetische Störungen aus der Umgebung ein (Motoren, Leuchtstoffröhren, Schaltnetzteile).
Jede Ader nimmt unterschiedlich starke Störungen auf.
Das digitale Signal wird verfälscht → Bitfehler, Synchronisationsverlust.

Lösung: Verdrillte Adernpaare (Twisted Pair)
Ethernet arbeitet differenziell:

Ein Signal wird gleichzeitig positiv und negativ auf zwei Adern gesendet.
Durch die regelmäßige Verdrillung:

liegen beide Adern immer wieder abwechselnd näher an einer Störquelle,
die eingekoppelte Störung ist auf beiden Drähten nahezu gleich groß.

![alt text](Designer.png)

## Physikalischer Effekt:
Am Empfänger werden die beiden Signale voneinander subtrahiert.

Nutzsignal bleibt übrig
Störung hebt sich auf (Gleichtaktunterdrückung / Common Mode Rejection)

Ohne Verdrillung funktioniert dieses Prinzip nicht → das Ethernet‑Signal bricht zusammen, besonders bei 15 m und CAT‑6‑Frequenzen (bis 250 MHz).

# 2. Welche Funktion hat der Aluminiumschirm (z. B. S/FTP)?
Aufgabe des Aluminiumschirms
Der Schirm wirkt wie ein Faradayscher Käfig und hat zwei Hauptfunktionen:
## Schutz vor externen Störungen (EMI):

Elektromotoren
Förderbänder
Gabelstapler
Schweißgeräte
Schaltnetzteile

## Verhinderung eigener Abstrahlung:

Das Kabel selbst sendet weniger Störungen aus
Wichtig bei paralleler Verlegung zu anderen Daten‑ oder Stromleitungen

Warum ist das im Lager besonders kritisch?
In Lagerhallen gibt es:

hohe Anlaufströme
starke Magnetfelder
lange Kabeltrassen

## Ungeschirmtes Kabel + keine Verdrillung = maximale Störanfälligkeit
Das Signal erreicht den Access Point zwar elektrisch, aber logisch unlesbar → der Tester zeigt „Fehler auf Layer 1“.

# 3. Skizze: Single‑Ended vs. differentiell
(Die Grafik oben kannst du direkt für Präsentation oder Prüfungserklärung verwenden.)
Kernaussage der Skizze:

Einzelader: Störung verändert das Signal direkt
Verdrilltes Aderpaar: Störung wirkt auf beide Adern gleich → wird herausgerechnet


# 4. Warum hätte ein 15‑cm‑Kabel evtl. trotzdem funktioniert?
Bei nur 15 cm Kabellänge:

ist die Antennenwirkung minimal,
koppeln kaum externe Störungen ein,
die Signalform bleibt trotz schlechter Kabelgeometrie noch ausreichend sauber.

## Kurz gesagt:
Je kürzer das Kabel, desto geringer die Störeinflüsse – bei 15 m sind Verdrillung und Schirm aber zwingend notwendig.

# Merksatz (IHK‑tauglich)

Verdrillung kompensiert elektromagnetische Störungen durch differentielle Signalübertragung, während der Aluminiumschirm externe Störeinflüsse abschirmt – beides ist bei längeren CAT‑Kabeln für eine fehlerfreie Übertragung auf OSI‑Schicht 1 unerlässlich.

# examensgerecht in 6–8 Stichpunkten (IHK‑Niveau):

- Ethernet nutzt differentielle Signalübertragung, bei der ein Signal über zwei verdrillte Adern gesendet wird.
- Die Verdrillung der Aderpaare sorgt dafür, dass elektromagnetische Störungen auf beide Adern nahezu gleich einwirken.
- Diese gleichartigen Störungen heben sich am Empfänger durch Differenzbildung (Gleichtaktunterdrückung) auf.
- Parallel geführte Einzeladern wirken dagegen wie Antennen und nehmen Störungen ungleichmäßig auf.
- Der Aluminiumschirm eines CAT‑Kabels schützt die Adern vor externen elektromagnetischen Einflüssen (EMI).
- Besonders in Lagerhallen mit Motoren und Maschinen ist Abschirmung wichtig, um Signalverfälschungen zu verhindern.
- Ohne Verdrillung und Schirmung ist bei längeren Kabeln keine zuverlässige Übertragung auf OSI‑Schicht 1 möglich.

# 🚨 5‑Sekunden‑Notfallantwort:

„Die MAC‑Adresse besteht aus Hersteller‑OUI und Geräte‑ID und identifiziert Geräte eindeutig auf Layer 2.“



# Beispiele für MAC‑Adressen & Dekodierung
# 🔹 Grundlagen (kurz)

* MAC‑Adresse = 48 Bit / 6 Byte / hexadezimal
* Aufbau: OUI (Hersteller) + Geräte‑ID
* OUI = erste 3 Byte (vom IEEE vergeben)
* MAC arbeitet auf OSI‑Schicht 2 


# 🧩 Beispiel 1: 00:1A:2B:3C:4D:5E
Dekodierung:

* OUI: 00:1A:2B → Cisco Systems (IEEE‑registrierter Hersteller) 
* Geräte‑ID: 3C:4D:5E → eindeutige Kennung des Geräts
* Adress‑Typ:

*           Unicast (I/G‑Bit = 0)
*           Universell administriert (U/L‑Bit = 0)



# ✅ Bedeutung:
Physische Netzwerkkarte eines Cisco‑Geräts (z. B. Switch, Router)

# 🧩 Beispiel 2: DC:A6:32:BE:1F:21
Dekodierung:

* OUI: DC:A6:32 → Apple, Inc. (z. B. iPhone, MacBook) 
* Geräte‑ID: BE:1F:21
* Adress‑Typ: Unicast, universell administriert

# ✅ Bedeutung:
Apple‑Endgerät im LAN oder WLAN

# 🧩 Beispiel 3: 02:42:AC:11:00:02
Dekodierung:

* OUI: 02:42:AC
* U/L‑Bit = 1 → lokal administrierte Adresse (LAA) 
* Typisch für virtuelle Maschinen / Container (Docker)

# ✅ Bedeutung:
Software‑vergebene MAC, nicht vom Hersteller fest vergeben

# 🧩 Beispiel 4: FF:FF:FF:FF:FF:FF
Dekodierung:

* Broadcast‑Adresse
* Erreicht alle Geräte im lokalen Netzwerksegment 

# ✅ Bedeutung:
z. B. ARP‑Anfragen im LAN

# 🧠 Gemeinsame Zusammenfassung (Ergebnis)

* MAC‑Adressen identifizieren Netzwerkgeräte eindeutig auf Layer 2
* Der Hersteller lässt sich über das OUI bestimmen
Es gibt:

*       Universelle MACs (physische Hardware)
*       Lokale MACs (virtuell / Software)


* Broadcast‑MAC wird für Netzwerkdienste genutzt
* MAC‑Adressen sind nur lokal gültig, nicht routbar

# 🎤 Mündliche Prüfungsfragen – MAC‑Adressen

# 1️⃣ Was ist eine MAC‑Adresse?
Musterantwort:

* „Eine MAC‑Adresse ist eine eindeutige 48‑Bit‑Hardwareadresse auf OSI‑Schicht 2, mit der Geräte im lokalen Netzwerk identifiziert werden.“ 


# 2️⃣ Wie ist eine MAC‑Adresse aufgebaut?
Musterantwort:

* „Sie besteht aus sechs Hexadezimal‑Bytes: den ersten drei für den Hersteller‑OUI und den letzten drei für die gerätespezifische ID.“ 


# 3️⃣ Was ist ein OUI?
Musterantwort:

* „Der OUI ist der Herstelleranteil einer MAC‑Adresse und wird von der IEEE vergeben, um den Gerätehersteller eindeutig zu identifizieren.“ 


# 4️⃣ Was bedeutet Unicast, Multicast und Broadcast bei MAC‑Adressen?
Musterantwort:

* „Unicast adressiert ein einzelnes Gerät, Multicast eine Gerätegruppe und Broadcast alle Geräte im lokalen Netzwerk.“ 


# 5️⃣ Woran erkennt man eine Broadcast‑MAC‑Adresse?
Musterantwort:

* „Die Broadcast‑Adresse lautet FF:FF:FF:FF:FF:FF und erreicht alle Geräte im selben Netzwerksegment.“ 


# 6️⃣ Was ist der Unterschied zwischen UAA und LAA?
Musterantwort:

* „UAA wird vom Hersteller vergeben, LAA ist lokal administriert und wird z. B. von Software oder virtuellen Maschinen gesetzt.“ 


# 7️⃣ Nennen Sie ein Beispiel für eine lokal administrierte MAC‑Adresse.
Musterantwort:

* „MAC‑Adressen von virtuellen Maschinen oder Docker‑Containern sind oft lokal administriert.“ 


# 8️⃣ Können MAC‑Adressen geroutet werden?
Musterantwort:

* „Nein, MAC‑Adressen gelten nur im lokalen Netzwerk und werden nicht über Router hinweg verwendet.“ 


# 9️⃣ Wie hilft die MAC‑Adresse bei der Netzwerkkommunikation?
Musterantwort:

* „Sie ermöglicht die eindeutige Zustellung von Frames im LAN, z. B. durch Switches.“ 


# 🔟 Wie kann man den Hersteller einer MAC‑Adresse ermitteln?
Musterantwort:

* „Über den OUI, indem man die ersten drei Bytes der MAC‑Adresse in einer IEEE‑OUI‑Datenbank nachschlägt.“ 


# 🚨 5‑Sekunden‑Notfallantwort (Bonus)

* „Die MAC‑Adresse identifiziert Geräte im LAN über Hersteller‑OUI und Geräte‑ID auf Layer 2.“
